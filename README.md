# Source Lens

**Browse the web through the lens of any primary source document.**

Source Lens is a Chrome extension that lets students carry a book, speech, legal document, or any text as a research companion while they browse the internet. As you read any web page, the source document responds — surfacing the most relevant passages in real time.

This is not AI generating opinions. It retrieves **real passages from real documents** based on the similarity of ideas. The source speaks for itself. The student does the thinking.

---

## Quick Start

You need [Node.js](https://nodejs.org/) installed (any recent version).

### Step 1: Add your document

Put a `.txt` or `.md` file in the `sources/` folder:

```
sources/
  the-republic.txt
  us-constitution.txt
  letter-from-birmingham-jail.txt
```

### Step 2: Build

```bash
node build.js sources/your-document.txt --title "The Title" --author "The Author"
```

That's it. The tool reads your document, splits it into searchable passages, and prepares the extension.

### Step 3: Load in Chrome

1. Open `chrome://extensions` in your browser
2. Turn on **Developer mode** (toggle in the top right)
3. Click **Load unpacked**
4. Select the `extension` folder inside `source-lens`
5. The Source Lens icon appears in your toolbar

### Step 4: Use it

1. Go to any website — a news article, Wikipedia, anything
2. Click the Source Lens icon → **Open Side Panel**
3. **Search**: Type a question to search the source document
4. **Page Scan**: Click the scan button — it reads the current page and finds relevant passages from your source
5. **Click any result** to read the full passage

## Examples

```bash
# A single document
node build.js sources/us-constitution.txt --title "US Constitution" --author "1787"

# A book in markdown
node build.js sources/walden.md --title "Walden" --author "Henry David Thoreau"

# Multiple documents at once (all .txt and .md files in a folder)
node build.js sources/ --title "Civil Rights Primary Sources" --author "Various"

# See what the tool does
node build.js --help
```

---

## What it does, technically

1. **Reads** your document and detects its structure (chapters, sections, headings)
2. **Splits** the text into overlapping passages (~600 characters each) so nothing falls between the cracks
3. **Creates vectors** — each passage becomes a list of 384 numbers that represent its meaning
4. **Saves** everything into the `extension/data/` folder
5. The Chrome extension **loads** these passages and vectors
6. When you search or scan a page, it compares the page's meaning against every passage and shows the closest matches

The passage retrieval takes 3-5 milliseconds. Everything runs locally in your browser. No data is sent anywhere.

## For Teachers

### Classroom ideas

- **History**: Load the Declaration of Independence. Have students browse modern news about government and see which passages resonate.
- **Literature**: Load a novel or poem. Students browse literary criticism and see what the original text actually says versus how critics interpret it.
- **Civics**: Load the Constitution. Students browse news about laws, court decisions, or policy debates with the actual constitutional text responding in the sidebar.
- **Philosophy**: Load primary texts (Plato, Locke, Marx, Wollstonecraft). Students browse contemporary issues and discover which ideas have deep roots.
- **Media Literacy**: Load any trusted primary source. Students browse news coverage and check: does the reporting match what the source actually says?

### What students learn

- How to work with primary sources directly, not summaries
- That ideas have histories — today's debates echo arguments made centuries ago
- To question framing — every news article carries assumptions that primary sources can surface
- That "what does the document actually say?" is always a valid question

### No AI hallucination

This tool does **not** generate text. It does not pretend to be a historical figure. It does not create opinions. It finds real passages from the actual document you loaded. Students see the real words. They can verify every result by reading the full passage.

## File structure

```
source-lens/
  build.js              ← The one command you run
  sources/              ← Drop your documents here
    example.txt
  extension/            ← Load this folder in Chrome
    manifest.json
    sidepanel.html/js   ← The side panel UI
    content.js          ← Extracts text from web pages
    background.js       ← Extension lifecycle
    lib/search-engine.js ← Vector search engine
    data/               ← Generated by build.js
      fragments.json    ← Your document's passages + metadata
      vectors.bin       ← Searchable vectors (binary, fast)
      manifest.json     ← Source info
    styles.css
    popup.html
    icons/
```

## Supported formats

| Format | Extension | Notes |
|--------|-----------|-------|
| Plain text | `.txt` | Any text file works |
| Markdown | `.md` | Headings become sections, formatting is stripped |

PDF support is planned. For now, copy-paste PDF text into a `.txt` file.

---

## Limitations

- The vector search finds topically related passages, not exact semantic matches. It's good at "this passage is about the same subject" but doesn't understand nuance the way a human reader does.
- Very short documents (under 1 page) produce few passages and limited results.
- The extension works in Chrome and Chromium-based browsers (Edge, Brave, Arc). Firefox does not support the Side Panel API yet.
